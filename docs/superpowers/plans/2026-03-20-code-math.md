# code_math Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `blog/code_math` as an independent Git-backed static site generator for paired code/math annotations, with an editorial homepage and article pages generated from self-contained Markdown articles.

**Architecture:** Use a small Node-based static generator. Parse article front matter and custom `:::pair` blocks into a normalized data model, render static HTML through focused template modules, and ship a thin client script only for pair highlighting and hash focus. Keep content loading entirely build-time.

**Tech Stack:** Node.js, built-in `node:test`, `gray-matter`, `markdown-it`, `markdown-it-katex`, plain CSS, plain browser JavaScript
